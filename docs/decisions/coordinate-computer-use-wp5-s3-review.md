## 2. 对照严谨性

**f-help-zh「命中」的巧合性质经数据证实**：其预测 (75,599) 即上述退化点（int8.json:105-106），与命令语义无关；gt 恰在左下 (76,602)（golden.json:1189-1193）故偏差 3.2px「命中」（int8.json:118-119）。报告自披露（REPORT.md:42）与数据一致。剔除巧合命中后，zh 真实能力仅 f-long-zh 1/15 ≈ 6.7%——比标题数字 13.3% **更差**，裁决方向不变（f-long-zh 两次命中偏差 46.2px/18.1px，真实落在 700px 宽目标区域内，属实打命中，见 §6）。

**en 组臂差来源可追溯**：en int8 50% vs fp32 25% 的全部差异来自单一 case f-ok-en（int8 3.6px 命中 int8.json:30-31；fp32 491.3px 未中 fp32.json:30-31）。n=4 下 1 case 翻转即 ±25%，报告定性「样本小」「argmax 掷币」（REPORT.md:32,34）准确，未过度解读。

**token 一致性**：int8 vs fp32 7 token 中 6 位逐位一致，唯一差 loc bin 284 vs 282（REPORT.md:18）——「量化不损伤输出语义」成立。细节：2 bin 在报告自述的 560px 图上实为 ≈1.1px（2/1000×560），报告写「2px」，量级无害（观察-2）。

## 4. 裁决边界（交叉 case 分析）

「命令语言 × 像素语言」的交叉对照天然内嵌于 golden 集：f-ok-en / f-play-en / f-icon-en 是**英文命令作用于中文夹具像素**（golden.json:1015-1019、:1237-1241、:1370-1374），与 zh case 共用同一图像与同一 gt 坐标系。这条交叉证据链完成了干净的因果分离：

- **中文像素可理解**：f-icon-en 双臂 1px 命中（int8.json:206-207；fp32.json:206-207）、f-ok-en int8 3.6px 命中（int8.json:30-31）——模型能看懂中文 UI 像素；
- **失效归因于命令语言**：同图、同 gt (884,602) 的 f-ok-zh 退化到 (75,600)、偏差 809px（int8.json:39-40, 52-53）——像素不变、命令换中文即崩；
- 结论「模型不理解中文命令语义（训练集 ScreenSpot 为英文）」（REPORT.md:42）的归因边界无混淆、无越界。

个例诚实度：f-play-en 两臂均误中蓝色图标 (481,171)、距播放按钮 ~150px（int8.json:127-128；fp32.json:127-128），报告自披露为「OOD 下的语义混淆个例」（REPORT.md:44），未粉饰英文臂 50%/25% 的真实成色。裁决边界核验结论：**FAIL 被严格限定在「中文 GUI 命令点定位 + 默认兜底资格」范围内，报告未把结论外推到「模型整体无用」**——英文命令约束下的实验层价值（REPORT.md:59）与 FAIL 裁决并存，边界划得正确。
