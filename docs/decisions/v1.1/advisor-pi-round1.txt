## Concrete failure modes

**(a) MV3 SW + `chrome.scripting.executeScript`:**
- SW 30s idle timeout + memory-pressure eviction kills in-flight batches; promise rejects with `Extension context invalidated`. Persist state to `chrome.storage.local` after *every* source, not at end.
- `func` arg cannot close over batch state — must be serializable; pass everything via `args`.
- `world: 'MAIN'` needed to poke Angular's `window.NgModel` if you ever do; default ISOLATED can't see page JS — easy footgun.
- Re-injecting on a tab already mid-dialog double-fires handlers. Track tab-state machine in SW, dedupe by `(tabId, dialogPhase)`.

**(b) SNlM0e CSRF:**
- Rotates on: logout/login, account switcher, long idle (hours, not minutes), server-side roll. **Not** per-request.
- Detect failure: `batchexecute` returns redirect to `accounts.google.com`, or HTTP 200 with empty `wrb.fr` envelope (silent invalidation — most common).
- Cache strategy: cache in `chrome.storage.local` with TTL, but **always re-extract on batch start**. Treat 0-notebook response as "token maybe dead, refetch HTML and retry once."
- Multi-account gotcha: user switches Google account in another tab → cached token silently invalid against new session.

**(c) Angular state desync (highest flake risk):**
- NotebookLM = Angular Material + zone.js. Click → element exists in DOM but `ngModel` not bound → you set `textarea.value` → Angular next CD cycle overwrites with `''`.
- Submit button `[disabled]` is Angular-controlled; clicking before enabled is silent no-op (no error, no add — worst kind of failure).
- Dialog enter-animation ~250ms; selector matches before `pointer-events:auto`.
- Source-ack `.single-source-container` renders async; diffing before CD runs falsely reports "not added" → infinite retry.
- **Mitigation (mandatory for v1.1):** after every action, wait via `MutationObserver` quiescence + `requestAnimationFrame` x2, **not** fixed `setTimeout`. Assert on Angular state (`disabled` attr, ng-pristine/dirty classes), not just selector existence. Without this you'll see ~30% flake on batches >10.
