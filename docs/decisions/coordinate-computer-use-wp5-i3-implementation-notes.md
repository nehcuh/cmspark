## ④ terminatedWorkers 取舍 + 首触 35s 时间线

### terminatedWorkers：Set 只增不删的取舍

**现状**（`companion/src/computer/tinyclick-runtime.ts:232`）：
`terminatedWorkers` 是 `Set<WorkerLike>`，worker 一旦进入（幂等闸 :517-518 / 故障首次入事件 :547-549）即永久驻留至进程结束，**永不删除**。

**取舍**：

| 选项 | 收益 | 代价 |
| --- | --- | --- |
| 只增不删（现状） | terminate/error 双路径天然幂等——同一 worker 的第二次事件 `has()` 即返，**熔断计数不会双计**（M6 熔断正确性 > 内存洁癖） | 已终止 worker 对象引用驻留至进程结束 |
| 定期清理 | 省引用 | 清理窗口内重到事件无法判重 → 双计风险；需要额外代际标记，复杂度换 negligible 内存 |

**量级评估**：worker 生命周期 = 懒加载一次 + 熔断/超时后重建。正常会话每进程 worker 总数为**个位数**（maxFaults 默认 3 即熔断停用）；最坏情况（反复 reset_circuit_breaker + 再故障）也只是每 reset 周期 +1。驻留代价为每实例一个 Map entry 引用，可忽略。**结论：维持只增不删，登记本取舍。**

### 首触 35s 时间线声明

- 代码事实：`loadTimeoutMs` 默认 **30000ms**（`tinyclick-runtime.ts:255`），冷启动加载超时 → terminate + 「不计熔断」处理（:400-402 日志原文「冷启动，不计熔断」——M6 纪律：加载期故障不进熔断计数）。
- 文案落点（WI-3.4 `f237b63` 已落）：`MODEL_SWITCH_COPY.firstLoadTimeline` =「模型首次加载最长约 35 秒（超时自动降级，且不计入故障熔断）；加载期间 UIA / OCR / 用户框选定位不受影响。」35s = 30s 超时 + 5s 余量（worker 启动/会话创建，hybrid 实测 createMs ~1.3-1.5s，余量充足）。
- 纪律：时间线文案只含上界声明，**不夹带未校准性能数字**（有测试强制：`computer-model-states.test.ts`「时间线文案」用例断言无 准确率/命中率/成功率 字样）。

## ⑥ 宽高比观测档登记

- **事实**（envelope §2.5 附记，I2 对抗 M6 补记）：宽高比是**未测定维**——S-3/G1 全部准确率领证于 3:2 夹具与被拒的 16:9@3840；竖屏 1080×1920（宽 ≤1920 放行）、21:9@1920、16:9@1920 均落包线内且**零准确率实测**；stretch 非等比缩放对竖屏的垂直挤压（~2.5×）是训练分布外形态。
- **I3 代码态**：包线只约束帧宽 >1920（`MAX_FRAME_WIDTH`），**未强制宽高比**——与 envelope 定案一致（文档级声明「包线通过 ≠ 命中承诺」+ reL2 人审兜底，不加代码围栏）。
- **登记结论**：按父指令「实测或文档注明」取**文档注明**——本档即观测档登记；宽高比实测补测留作后续迭代候选（需竖屏/超宽采集夹具，当前无数据集，不伪造实测）。
