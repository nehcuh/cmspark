# CMspark Browser Agent — v2 产品需求

## R12 — Type B Skills（工具链/流程执行引擎）

### R12.1 工具链定义
- Skill markdown frontmatter 中添加 `steps` 字段（YAML 结构化步骤）
- 每步定义：tool, params, condition, on_error
- 参数化占位符：`{{param_name}}`

### R12.2 执行引擎
- 按序执行 steps
- 条件分支（condition 判断）
- 步骤间状态传递
- 执行中暂停/继续/跳过

### R12.3 参数提示
- 执行前提示用户填写 `required` 参数
- 可选参数使用默认值
- 参数支持类型校验（string, number, boolean, enum）

---

## R13 — Type C Skills（子 Agent）

### R13.1 子 Agent 创建
- 主 Agent 在 tool call 中创建子 Agent
- 配置：system prompt + 工具白名单 + 超时时间
- 并发上限 3 个

### R13.2 子 Agent 执行
- 独立 LLM context（不污染主线程）
- 权限继承（可降级，如"只读模式"）
- 超时 120s 自动终止

### R13.3 结果返回
- 摘要返回主 Agent（不返回完整 context）
- 失败报告（含原因）

## R14 — "保存对话为 Skill" 核心流程

### R14.1 事后分析
- 用户在线程中选中消息范围
- Agent 分析 tool call 序列
- 识别可复用的操作模式

### R14.2 参数化
- LLM 自动识别硬编码值的语义角色
- 建议参数名和类型
- 用户确认/调整 → 生成 skill 文件

### R14.3 测试与验证
- 生成后提供"测试运行"按钮
- 用测试参数试运行验证

## R15 — 操作历史增强

### R15.1 历史重放
- 从历史记录中选择操作点
- 从该点重新执行（重新创建 thread context）

### R15.2 高级筛选
- 按日期范围、工具类型、线程、成功/失败筛选
- 导出 CSV

### R16.1 Daemon 模式

### R16.2 远程安装 Skill
