# CMspark — 技能系统增强需求

> 版本: 1.0.0 | 日期: 2026-05-26 | 状态: 待确认

---

## 背景

当前技能激活方式只能通过 BottomBar Skills 面板勾选 checkbox，操作路径长，不符合对话式交互习惯。架构文档中已规划 `/skill-name` 内联调用语法和"保存对话为 Skill"流程，但均未实现。

本次增强聚焦两个核心能力：
- **F1 — 反斜线命令弹窗**：输入 `/` 触发技能补全，快速调用
- **F2 — 技能提取**：从对话历史中分析提取可复用技能

---

## F1 — 反斜线命令弹窗（Slash Command Popover）

### F1.1 触发机制

| 场景 | 行为 |
|------|------|
| 用户在输入框行首或空格后输入 `/` | 弹出候选技能列表 |
| 用户继续输入文字 | 实时过滤匹配的技能 |
| 用户按下 Esc | 关闭弹窗 |
| 用户按下 ↑↓ | 在候选列表中移动高亮 |
| 用户按下 Enter | 选中高亮项，插入命令文本 |
| 用户点击弹窗外的区域 | 关闭弹窗 |
| 用户删除 `/` 前缀 | 弹窗关闭 |
| 用户在非行首位置输入 `/` | 不触发弹窗（仅行首或空格后触发） |

### F1.2 弹窗 UI

```
┌──────────────────────────────────────┐
│  /brow                               │  ← textarea
│                                      │
│  ┌────────────────────────────────┐  │
│  │ 🔧 /browse                     │  │  ← popover（绝对定位）
│  │    浏览器操作指南 — 指导 Agent  │  │     高亮匹配字符
│  │    如何有效控制浏览器           │  │
│  │                                │  │
│  │ 📋 /grill-me                   │  │
│  │    设计审查对话 — 对代码变更    │  │
│  │    进行结构化设计审查          │  │
│  │                                │  │
│  │ ✏️ /writing-skills             │  │
│  │    Skill 创建方法论 — 指导如何  │  │
│  │    编写高质量技能文件          │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

### F1.3 弹窗规格

| 属性 | 值 |
|------|-----|
| 宽度 | 与 textarea 等宽 |
| 最大高度 | 240px，超出滚动 |
| 位置 | textarea 上方弹出（空间不足时下方） |
| 每项高度 | 56px（名称 + 描述两行） |
| 高亮色 | `#E8F0FE` 背景 |
| 匹配文字 | 加粗或高亮 |
| 类型图标 | builtin: 🔧, user: 📋, 按 skill type 区分 |
| 无匹配时 | 显示 "无匹配技能" |

### F1.4 匹配规则

- 模糊匹配：skill name 和 description 均参与匹配
- 匹配优先级：
  1. name 前缀精确匹配（`/bro` → `/browse`）
  2. name 包含匹配（`/row` → `/browse`）
  3. description 包含匹配
- 匹配部分在 UI 中高亮显示

### F1.5 命令发送

```
用户输入: "/browse 帮我打开百度"
                │
                ▼
         发送前解析:
           command: "browse"
           args: "帮我打开百度"
           skillId: "browse" (从 skills 列表匹配)
                │
                ▼
         chat.send 携带:
           message: "/browse 帮我打开百度"  (保留命令文本)
           skillIds: ["browse"]             (自动激活)
```

### F1.6 边界情况

| 场景 | 行为 |
|------|------|
| 已激活技能重复调用 | 不重复激活，正常发送消息 |
| 技能名不存在 | 发送普通消息，不激活任何技能 |
| 多个 `/` 命令 | 仅识别第一个（或全部识别，TBD） |
| 文本中非命令 `/` | 不触发补全（仅行首/空格后触发） |
| 技能列表为空 | 弹窗不显示 |
| 输入框为空只有 `/` | 显示全部技能列表 |

## F2 — 技能提取（Skill-craft / 保存对话为 Skill）

### F2.1 触发入口

| 入口 | 位置 |
|------|------|
| 主入口 | 线程顶部菜单 → "提取技能" 按钮 |
| 快捷入口 | 消息区域选中多条消息后出现的浮动操作栏 |
| 命令入口 | 输入 `/skill-craft` 命令（F1 实现后） |

### F2.2 提取流程

```
用户触发 "提取技能"
        │
        ▼
┌──────────────────────────┐
│ 1. 选择分析范围          │
│    ○ 整个线程对话         │
│    ○ 选中的消息范围       │
│    ○ 最近 N 轮对话        │
└──────────┬───────────────┘
           │
           ▼
┌──────────────────────────┐
│ 2. 发送分析请求          │
│    → Companion LLM 分析   │
│    显示 loading 状态      │
│    "正在分析对话历史..."  │
└──────────┬───────────────┘
           │
           ▼
┌──────────────────────────┐
│ 3. 预览生成的 Skill       │
│    ┌──────────────────┐  │
│    │ name: ...         │  │
│    │ description: ...  │  │
│    │ steps:            │  │
│    │   - tool: ...     │  │
│    │     params: ...   │  │
│    └──────────────────┘  │
│    [测试运行] [编辑] [保存]│
└──────────┬───────────────┘
           │
           ▼
┌──────────────────────────┐
│ 4. 保存技能              │
│    → ~/.cmspark-agent/   │
│      skills/<name>.md    │
│    → 刷新技能列表         │
│    → 通知用户             │
└──────────────────────────┘
```

### F2.3 分析 Prompt 设计

LLM 分析时使用的 system prompt 核心指令：

```
你是一个技能提取器。分析以下对话历史中的 Agent 操作，
识别可复用的操作模式。

要求：
1. 提取 tool call 序列作为技能步骤
2. 识别硬编码值中的参数（URL、搜索词、日期等）
3. 为技能生成简洁的名称和描述
4. 输出格式：Markdown + YAML frontmatter

技能类型：
- prompt_template: 纯文本指导（没有明确操作序列时）
- tool_chain: 有明确步骤序列时，使用 YAML steps 定义
```

### F2.4 生成的 Skill 格式

```markdown
---
name: export-report
description: 从内部报表系统导出周期性数据
type: tool_chain
parameters:
  system_url:
    type: string
    required: true
    description: 报表系统地址
  report_type:
    type: string
    default: "销售报表"
    description: 报表类型
  date_range:
    type: string
    default: "上周"
    description: 日期范围
---

# 导出报表

1. 导航到 {{system_url}}
2. 如果未登录，使用 SSO 自动登录
3. 点击 "{{report_type}}"
4. 选择日期范围: {{date_range}}
5. 点击导出，下载文件
```

### F2.5 预览编辑面板

| 字段 | 可编辑 | 说明 |
|------|--------|------|
| name | 是 | 技能标识符（字母、数字、连字符） |
| description | 是 | 一句话描述 |
| type | 是 | prompt_template / tool_chain |
| parameters | 是 | 参数定义，可增删改 |
| body | 是 | Markdown 正文，自由编辑 |

### F2.6 离线提示

由于分析依赖 LLM 调用，用户当前配置的 LLM 将用于技能提取。建议在分析期间保持连接。

## F3 — 公共约束

### F3.1 架构约束

| 约束 | 说明 |
|------|------|
| F1 前端实现 | Slash popover 纯 Extension 端实现，不依赖 companion |
